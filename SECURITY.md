# Security Policy

## Supported versions

Security fixes are provided for the latest released version of qc2.

## Reporting a vulnerability

Use GitHub private vulnerability reporting for this repository when possible.

If private reporting is unavailable, open an issue containing only a brief description and a way to contact you. Do not include exploit details, secrets, or sensitive system information in a public issue.

You can expect an initial response within seven days. Confirmed vulnerabilities will be fixed and released as soon as practical.
